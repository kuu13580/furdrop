# ストレージコスト試算

## 前提条件

| 項目 | 値 |
|---|---|
| 受信者数 | 100人（初期最大） |
| 1受信者あたりの写真枚数 | 1,000枚 |
| 1枚あたりの平均サイズ（JPG, 一眼レフ） | 10MB |
| 総写真枚数 | 100,000枚 |
| 総ストレージ容量 | **1TB** |
| サムネイル（長辺300px, 低品質JPG） | ~30KB/枚 → 合計 ~3GB（無視できるレベル） |
| リージョン | us-east-1（東京より安いため基準値として使用） |

## Intelligent-Tiering（推奨候補）

S3 Intelligent-Tieringは**復元料金が無料**で、アクセスパターンに応じて自動的にストレージ層を移動します。

| アクセス層 | $/GB/月 | 条件 |
|---|---|---|
| Frequent Access | $0.023 | アクセスがあった場合 |
| Infrequent Access | $0.0125 | 30日間未アクセス |
| Archive Instant Access | $0.004 | 90日間未アクセス |

**追加コスト**: モニタリング料 $0.0025/1,000オブジェクト = 100,000枚で **$0.25/月**

### Intelligent-Tieringのシミュレーション

写真は受信後しばらくは閲覧されるが、時間が経つと大半はアクセスされなくなる想定:

| 期間 | 写真の状態 | 適用層 | コスト |
|---|---|---|---|
| 受信後0-30日 | アクティブに閲覧 | Frequent ($0.023) | 通常料金 |
| 30-90日後 | ほぼ放置 | Infrequent ($0.0125) | 約46%削減 |
| 90日以降 | 完全放置 | Archive Instant ($0.004) | 約83%削減 |

**定常状態（大半が90日以上経過）の月額**:
- 1TB × $0.004 + モニタリング$0.25 ≈ **$4.25/月**
- **復元料金: $0**（どの層からアクセスしても無料）
- **待ち時間: なし**（Archive Instant Accessはミリ秒応答）

## アップロードコスト

写真がアップロードされた際に発生するコスト。

### データ転送IN（アップロード帯域）

| プロバイダ | インバウンド転送料 |
|---|---|
| S3（全クラス共通） | **$0**（無料） |
| Cloudflare R2 | **$0**（無料） |

→ 1TB分のアップロード転送料は **どちらも$0**。

### PUTリクエスト料金（書き込み）

1枚のアップロードで最低2回のPUT（オリジナル + サムネイル）が発生。
100,000枚 × 2 = **200,000 PUT**

| ストレージクラス | PUT単価 ($/1K件) | 200K PUTの費用 | 備考 |
|---|---|---|---|
| Standard | $0.005 | **$1.00** | |
| One Zone-IA | $0.01 | **$2.00** | |
| Glacier Instant | $0.02 | **$4.00** | PUTが4倍高い |
| Glacier Flexible | $0.03 | **$6.00** | PUTが6倍高い |
| Intelligent-Tiering | $0.005 | **$1.00** | Standardと同額 |
| **Cloudflare R2** | $4.50/100万件 | **$0.90** | Class A operation |

→ アップロードのPUTコストは全100,000枚分でも **$1〜$6の一括コスト**（月額ではなく発生時のみ）。
→ 月間の新規アップロードが1,000枚（100受信者が月10枚ずつ受信）の場合、月$0.01〜$0.06程度で**無視できるレベル**。

### サムネイル生成コスト（サーバーサイドの場合）

サーバーサイドでサムネイルを生成する場合、コンピューティングコストが発生。

#### AWS Lambda で生成する場合

| 項目 | 値 |
|---|---|
| 1枚あたりの処理時間（10MB JPGリサイズ） | ~500ms |
| メモリ割り当て | 512MB |
| 1枚あたりのコスト | ~$0.000004 |
| 100,000枚の合計 | **$0.42** |
| 月1,000枚の場合 | **$0.004/月** |

#### Cloudflare Workers で生成する場合

| 項目 | 値 |
|---|---|
| 無料枠 | 100,000リクエスト/日 |
| 有料プラン（$5/月） | 10,000,000リクエスト/月 |
| 画像リサイズ（Cloudflare Images Transformations） | $0.50/1,000変換 |
| 100,000枚の合計 | **$50**（一括） |
| 月1,000枚の場合 | **$0.50/月** |

→ Lambda のほうがサムネイル生成は安い。ただしクライアントサイドで生成すれば**$0**。

### アップロードコストまとめ

| 項目 | S3 Intelligent-Tiering | Cloudflare R2 |
|---|---|---|
| データ転送IN | $0 | $0 |
| PUT 200K件 | $1.00 | $0.90 |
| サムネイル生成（Lambda/クライアント） | $0〜$0.42 | $0〜$50 |
| **アップロード合計（100K枚）** | **$1.00〜$1.42** | **$0.90〜$50.90** |
| **月間（1K枚新規）** | **$0.01〜$0.004** | **$0.0045〜$0.50** |

**結論: アップロードコストは全方式ともに極めて小さく、ストレージ・転送コストと比較して無視できるレベル。サムネイル生成はクライアントサイドにすればコスト$0。**

## 結論・推奨

### 推奨: **S3 Intelligent-Tiering**

| 観点 | 評価 |
|---|---|
| 月額コスト（定常状態） | ~$4.25（Glacierとほぼ同等） |
| 復元コスト | **$0**（最大の利点） |
| 復元速度 | ミリ秒（Standard同等） |
| アーキテクチャの複雑さ | **シンプル**（復元処理不要） |
| DL率の変動に強いか | **強い**（DL率に関係なく復元無料） |

### 比較まとめ

| 方式 | 月額目安 | 開発コスト | UX | 推奨度 |
|---|---|---|---|---|
| Standard一本 | $23 | 最小 | 最高 | DL率が高い場合に有利 |
| **Intelligent-Tiering** | **$4-23** | **最小** | **最高** | **総合的にベスト** |
| Glacier + 復元フロー | $4-20 | 高（復元処理実装必要） | 低（待ち時間あり） | 非推奨 |
| 当初案（低コスト→高速に移動） | $4-10 | 最高（2層管理+移動処理） | 中 | 非推奨 |

**Intelligent-Tieringは、当初ご提案の2層ストレージ戦略の目的（低コスト保存＋必要時に即座にアクセス）を、開発なしで自動的に実現します。**

## 補足: データ転送料金（全方式共通）

S3からインターネットへのデータ転送（ダウンロード）:
- 最初の100GB/月: $0.09/GB
- 100受信者が各5GB DLした場合: 500GB × $0.09 = **$45/月**

→ ストレージ料金より**転送料金のほうが高額になる**可能性がある点に注意。
→ CloudFront経由だと$0.085/GBだがキャッシュで削減可能。
→ または**Cloudflare R2**（転送料金$0）も検討候補。

### Cloudflare R2との比較

| | S3 Intelligent-Tiering | Cloudflare R2 |
|---|---|---|
| ストレージ | $0.004-0.023/GB/月 | $0.015/GB/月 |
| 1TB月額 | $4-23 | $15 |
| データ転送（egress） | $0.09/GB | **$0** |
| DL 500GB/月の転送料 | $45 | **$0** |
| **合計（定常+DL500GB）** | **$49-68** | **$15** |
| 自動階層化 | あり | なし |
| Glacier相当の低コスト層 | あり | なし（Infrequent Accessのみ） |

**DLが多い場合、Cloudflare R2のほうがトータルコストが安くなる可能性が高い。**
